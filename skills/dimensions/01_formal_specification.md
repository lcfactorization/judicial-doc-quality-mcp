# 形式规范 — 评分标准

## 评估哲学

> [!NOTE]
> 形式规范是裁判文书的"门面"。虽然本维度权重最低（3%），但形式缺陷往往影响文书的严肃性和可读性，严重的形式错误（如案号错误、当事人信息缺失）甚至可能影响裁判效力。本维度采用"底线思维"：基本合格即可，不追求完美。

**核心认知**：中国司法实践中，形式规范问题普遍存在但影响有限。最高人民法院2024年审判质量管理指标体系改革更关注实质质量而非形式瑕疵。因此，本维度权重设为最低，仅对严重形式错误给予显著扣分。

**参考框架**：最高法院优秀裁判文书标准研究（2012年评查指导标准）中的文书格式规范要求，以及三种新文书格式（要素式、令状式、表格式）的格式要求。

**民商事专项法律依据**：
- 《人民法院民事裁判文书制作规范》（法〔2016〕221号，2016年8月1日施行）
- 《最高人民法院关于人民法院案件案号的若干规定》
- 《最高人民法院关于裁判文书引用法律、法规等规范性法律文件的规定》（法释〔2009〕14号）
- 《民事诉讼法》第40条（合议庭）、第127条（管辖异议）、第137条（公开审理）、第146条（延期）、第150条（中止）、第163条（程序转换）、第164条（上诉权）、第166条（上诉期）

## 评分规则（满分 100 分，起评 100 分，扣分制）

### 一、扣分项

| 编号 | 扣分项 | 扣分值 | 判定标准 |
|:---|:---|:---|:---|
| F-01 | 法院名称或案号错误 | -25 | 法院名称不准确、案号格式错误或与实际不符 |
| F-02 | 当事人信息缺失或错误 | -20 | 当事人姓名/名称、住所地、性别、职务等基本信息缺失或错误 |
| F-03 | 案件由来表述不清 | -10 | 立案时间、案由、审理经过等表述过于简略或缺失 |
| F-04 | 审理经过不完整 | -10 | 未说明独任/合议庭、公开/不公开、开庭时间等 |
| F-04a | 程序转换未交代 | -5 | 简易程序转普通程序、追加当事人等程序转换事项未说明 |
| F-04b | 管辖异议处理未交代 | -5 | 当事人提出管辖异议但文书未交代处理结果 |
| F-04c | 保全措施未交代 | -3 | 案件涉及保全措施但文书未交代保全情况 |
| F-05 | 尾部告知事项缺失 | -8 | 上诉权利、期间、法院等告知事项缺失或不完整 |
| F-06 | 署名或印章不规范 | -5 | 审判人员署名、法院印章等不规范 |
| F-07 | 诉讼主体信息不完整 | -8 | 委托代理人信息缺失，当事人到庭情况不明 |
| F-08 | 文书结构不完整 | -10 | 缺少必要的文书组成部分（如首部、正文、尾部） |
| F-09 | 法条援引格式不规范 | -5 | 法条名称不准确、条款号错误、引用内容不完整 |
| F-10 | 数字或标点使用不规范 | -3 | 金额数字格式错误、标点符号使用不规范 |

### 二、加分项

| 编号 | 加分项 | 加分值 | 判定标准 |
|:---|:---|:---|:---|
| F-B01 | 信息完整准确 | +3 | 所有首部信息完整、准确、规范 |
| F-B02 | 审理经过详细透明 | +3 | 受理、开庭、审结等过程完整透明 |
| F-B03 | 采用新型文书格式 | +5 | 根据案件类型合理采用要素式/令状式/表格式文书格式 |

### 三、三种新文书格式参考

> [!NOTE]
> 以下为最高法院推广的三种新文书格式，适用于不同类型案件。采用合适的新型格式可获加分。

| 格式类型 | 适用场景 | 特点 |
|:---|:---|:---|
| 要素式 | 劳动争议、交通事故等类型化案件 | 夹叙夹议，围绕争议要素展开 |
| 令状式 | 20万以下、事实无争议、当庭宣判案件 | 只含诉讼参与人称谓和裁判主文 |
| 表格式 | 要素可用表格列举的案件 | 用表格列举方式陈述 |

### 四、民商事专项：审理经过必须交代的事项

> [!NOTE]
> 以下为《人民法院民事裁判文书制作规范》及《民事诉讼法》要求民商事裁判文书在审理经过部分必须交代的事项，遗漏任何一项均应扣分。

| 事项 | 规范要求 | 法律依据 |
|:---|:---|:---|
| 立案时间 | 应写明立案日期 | 《民事裁判文书制作规范》 |
| 合议庭/独任 | 写明组成人员或独任审判员 | 《民事诉讼法》第40条 |
| 公开/不公开 | 应说明是否公开审理及理由 | 《民事诉讼法》第137条 |
| 开庭时间 | 应写明开庭日期 | 《民事裁判文书制作规范》 |
| 程序转换 | 简易转普通应说明理由和时间 | 《民事诉讼法》第163条 |
| 管辖异议 | 应交代处理结果 | 《民事诉讼法》第127条 |
| 保全措施 | 应交代保全情况 | 《民事诉讼法》第100-105条 |
| 鉴定/评估 | 应交代委托鉴定/评估情况 | 《民事诉讼法》第76条 |
| 延期审理 | 应交代延期原因和时间 | 《民事诉讼法》第146条 |
| 中止/恢复 | 应交代中止和恢复诉讼的情况 | 《民事诉讼法》第150条 |

### 五、民商事专项：法条引用格式规范

> [!NOTE]
> 以下为法释〔2009〕14号及《民事裁判文书制作规范》关于法条引用的格式要求。

| 引用要素 | 规范要求 | 常见错误 |
|:---|:---|:---|
| 法律名称 | 应写全称 | "合同法"→应为"《中华人民共和国民法典》" |
| 条款号 | 应写明编、章、节、条、款、项 | "第58条"→应写明具体款、项 |
| 引用顺序 | 法律及法律解释→行政法规→地方性法规→自治条例→司法解释 | 顺序混乱 |
| 实体与程序 | 实体法与程序法并用时先实体后程序 | 先程序后实体 |
| 整条引用 | 以整条引用为宜 | 仅引用条款号未引用内容 |

请严格按照以下 JSON 格式输出评分结果：

```json
{
    "quote": "从文书中摘录一处最能体现形式规范质量（好或差）的原文段落，不超过200字",
    "reasoning": "一句话概括评分理由，包含主要扣分项或加分项",
    "score": 85,
    "deduction_items": [
        {
            "item": "扣分项名称（对应F-01至F-10编号）",
            "deduction": 10,
            "quote": "原文引用",
            "basis": "判定依据"
        }
    ],
    "bonus_items": [
        {
            "item": "加分项名称（对应F-B01至F-B03编号）",
            "bonus": 3,
            "quote": "原文引用",
            "reason": "加分理由"
        }
    ]
}
```

**重要约束**：
1. score 必须为 0-100 之间的整数
2. 扣分项和加分项必须有原文引用（quote字段），不得凭空判断
3. 同一问题不重复扣分（就高扣分）
4. 加分后总分不超过100分
5. 本维度权重低，形式瑕疵不应过度扣分，重点关注影响裁判效力的严重错误
